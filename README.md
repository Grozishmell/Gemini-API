# База знаний на векторной БД (RAG + ChromaDB)

Веб-приложение, которое отвечает на вопросы по коллекции загруженных PDF.
Несколько файлов хранятся в **векторной базе данных ChromaDB**: данные
сохраняются на диск, а поиск идёт сразу по всем документам.

Загрузите один или несколько PDF, и задавайте вопросы обычными словами — приложение
найдёт ответ в нужном документе, сформулирует его и покажет, из какого файла он взят.
Если ответа в базе нет, приложение честно об этом сообщит.

## Возможности

- Загрузка нескольких PDF одновременно
- Постоянное хранение: база переживает перезапуск приложения
- Поиск по **смыслу** сразу по всем документам
- Указание источника для каждого ответа
- Защита от «галлюцинаций» (ответы строго по базе)
- Список загруженных документов на боковой панели

## Технологии

- **Python**
- **Streamlit** — веб-интерфейс
- **ChromaDB** — векторная база данных (хранение и поиск эмбеддингов)
- **Google Gemini API** — генерация ответов (`gemini-3.5-flash`) и эмбеддинги (`gemini-embedding-001`)
- **pypdf** — чтение PDF

## Как это работает

Приложение реализует RAG-пайплайн поверх векторной базы данных:

1. **Чтение** — из каждого PDF извлекается текст.
2. **Нарезка** — текст делится на небольшие пересекающиеся кусочки (чанки).
3. **Эмбеддинги** — каждый кусочек превращается в вектор через Gemini.
4. **Хранение** — векторы, тексты и метаданные (имя документа) сохраняются в ChromaDB на диск.
5. **Поиск** — вопрос превращается в вектор, и ChromaDB находит ближайшие по смыслу кусочки.
6. **Генерация** — найденные кусочки передаются модели как контекст, и она отвечает,
   опираясь только на них, с указанием источника.

Повторная индексация одного и того же файла исключена: каждый документ помечается
именем в метаданных, и при загрузке уже известного файла приложение его пропускает.

## Установка

```bash
git clone https://github.com/ВАШ_ЛОГИН/knowledge-base-rag.git
cd knowledge-base-rag
pip install -r requirements.txt
```

## Настройка API-ключа

1. Получите бесплатный ключ в [Google AI Studio](https://aistudio.google.com).
2. Создайте файл `.streamlit/secrets.toml`:

```toml
GEMINI_API_KEY = "ваш_ключ"
```

> Файл с ключом не попадёт в репозиторий — он указан в `.gitignore`.

## Запуск

```bash
streamlit run knowledge_base.py
```

Откроется браузер. Загрузите PDF и задавайте вопросы.

## Структура проекта


## Примечание о хранении

База ChromaDB хранится локально в папке `knowledge_db/`. При развёртывании на
бесплатном облачном хостинге файловая система может сбрасываться при перезапуске —
в этом случае документы потребуется загрузить заново. Для постоянного хранения
в продакшене используется облачная векторная БД.
